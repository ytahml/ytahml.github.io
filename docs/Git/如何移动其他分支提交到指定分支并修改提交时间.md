# 如何移动其他分支提交到指定分支并修改提交时间

## 背景

在实际开发过程中, 我们需要创建新分支, 开发结束后再将分支合并到测试、测试通过后再合并到mater进行上线;

但是由于工作安排等原因, 有一些完成的代码没有及时部署上线, 此时再将遗留代码直接合并会存在一定问题: _直接变基、合并可能会有冲突,产生的提交记录不符合开发要求规范_;

如果采用直接将其他分支的代码复制粘贴到当前新分支, 当改动较小时还好, 若改动较大、涉及文件较大只靠复制粘贴就很麻烦, 因此需要采用 git 命令来快速完成这个操作;

## 方法一

通过以下步骤完成提交从分支1到分支2的迁移: 

```shell

# 执行 cheery-pick
git cheery-pick [需要迁移的提交哈希值]

# 修改最近一次提交的时间
git commit --amend --date="新的日期" --no-edit

# 需要可以修改作者日期
git commit --amend --reset-author --date="新的日期" --no-edit

```

## 方法二

直接执行如下命令完成分支的迁移和提交时间的修改

```shell
# GIT_COMMITTER_DATE 和 --date 可以分别设置提交者日期和作者日期

GIT_COMMITTER_DATE="新的日期" git cherry-pick 提交哈希值 --no-commit
GIT_COMMITTER_DATE="新的日期" git commit -m "提交描述" --date="新的日期"

```

如果有多个提交需要迁移, 只需要按照从旧到新的顺序,依次执行迁移命令即可
